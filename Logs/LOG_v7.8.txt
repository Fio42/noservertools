CAMBIOS v7.8
════════════════════════════════════════════════════════════

1. COMPRESS — REESCRITURA COMPLETA
   Problema raíz del freeze: runPythonAsync corre en el main thread de JS.
   Con PDFs grandes (>10MB) bloquea el event loop del browser — la página
   se congela completamente hasta que termina.

   SOLUCIÓN: Web Worker (background thread)
   - Todo Pyodide corre dentro de un Worker creado vía Blob URL
   - El main thread (UI) nunca bloquea
   - Comunicación via postMessage con transferencia zero-copy del ArrayBuffer
   - El buffer se transfiere al worker (no se copia) para máxima eficiencia

   REGLA #35: Pyodide + procesamiento pesado SIEMPRE en Web Worker.
   Nunca usar runPythonAsync directamente en el main thread para archivos grandes.

2. COMPRESS — 5 NIVELES DE COMPRESIÓN (antes 3)
   SCREEN    15% quality  → Smallest possible. Web/email.
   EBOOK     30% quality  → Aggressive. Good for sharing.
   BALANCED  50% quality  → Recommended default. (antes era 60%)
   HIGH      72% quality  → Sharp images. Good visuals.
   PRINT     88% quality  → Near lossless. Print ready.

   Calidades más agresivas en los niveles bajos para mejor compresión real.
   El algoritmo usa PIL para re-renderizar cada imagen embebida — más efectivo
   que el método anterior que solo usaba compress_identical_objects.

3. COMPRESS — LÍMITE SUBIDO A 200MB (antes 100MB)

4. NAV/FOOTER — Confirmado completo en todos los archivos
   Todos los 12 archivos tienen los 10 links en nav y footer.
   delete-pages y reorder-pages ya tenían todos los links correctos.
