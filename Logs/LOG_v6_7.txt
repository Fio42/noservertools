EDIT-PDF — SISTEMA DE TEXTO v6.7 (REESCRITO)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Flujo del usuario:
  1. Seleccionar herramienta "Text"
  2. Hacer clic en el PDF donde quiere el texto
  3. El sistema crea el text-box Y cambia automaticamente a modo Select
  4. El usuario puede arrastrar la barra azul para mover el texto
  5. El boton x en la barra elimina el text-box
  6. En modo Draw/Highlight puede seguir dibujando normalmente

Estructura DOM de cada texto:
  <div class="text-box" style="left:Xpx;top:Ypx">     <- wrapper posicionado
    <div class="text-box-bar">                          <- barra azul draggable
      <span class="text-box-bar-icon">☰ drag to move</span>
      <button class="text-box-del">x</button>
    </div>
    <textarea class="text-input-overlay">...</textarea>
  </div>

Modo select — clave del funcionamiento:
  annCanvas.style.pointerEvents = 'none'
  Esto hace que los clicks pasen ATRAVES del canvas transparente y lleguen
  a los .text-box que estan debajo en el DOM.
  En cualquier otro modo: pointerEvents = 'auto' (canvas intercepta todo)

makeDraggableBox(box, handle):
  Drag via la barra (.text-box-bar) solamente.
  Usa document.addEventListener para mousemove/mouseup (funciona fuera del elemento).

textInputs[pageNum] = array de divs .text-box (NO textareas directamente)

downloadPDF():
  Para cada box: box.querySelector('textarea') para obtener el texto
  Offset Y: parseFloat(box.style.top) + BAR_H (22px) para compensar la barra

NUNCA cambiar BAR_H sin actualizar el offset en downloadPDF.
